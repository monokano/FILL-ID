Permissions and Troubleshooting

Automation (Apple Events) permission

To control InDesign, FILL Id needs Automation (Apple Events) permission. The first time you run clean paste or another command, a dialog asks for permission; choose "OK."

If permission has not been granted, the menu bar and Dock menus show "⚠️ Automation permission required (InDesign)." Clicking it opens "System Settings › Privacy & Security › Automation." Turn on the item for FILL Id controlling InDesign.

Tip Automation permission is recorded per pair of "FILL Id to InDesign." Because all versions of InDesign share the same identifier, granting it once is expected to apply to every version.
Note FILL Id does not require Accessibility permission.

When things don't work

No hotkeys work at all
Check that InDesign is frontmost. FILL Id's hotkeys are active only while InDesign is frontmost.
Only a specific feature doesn't work
Under "Features" in Settings, check that the feature is on. For toggling grids and guides, "Enable display toggle" in Document Display Settings must be on.
Clean paste or zoom doesn't work
Check the Automation permission above. Also, the shortcuts for Fit and Zoom use the numeric keypad keys 0, +, and −.
The menu bar icon is missing
The display mode may be set to "Dock Only," or the icon may be hidden behind the notch. Hold the ⌘ key while starting FILL Id to reset the display mode to the initial state (Menu Bar and Dock).
